pwd (Print Working Directory): Utilizado para verificar em qual diretório eu estava trabalhando no momento, garantindo que os arquivos HTML e CSS fossem criados no local correto.

mkdir (Make Directory): Essencial para criar a estrutura de pastas do projeto, como a pasta css para organizar as folhas de estilo.

cp (Copy): Comando usado para realizar cópias de arquivos ou backups de segurança antes de fazer alterações críticas nos códigos.

mv (Move/Rename): Utilizado tanto para mover arquivos entre pastas quanto para renomear arquivos, como ajustar o nome de um arquivo .html ou .css.

ls (List): (Adicional) Frequentemente usado para listar os arquivos e confirmar se as pastas e códigos foram criados com sucesso.


Níveis de Dificuldade:
As principais dificuldades residem na curva de aprendizado dos editores de texto via terminal, especialmente no Vim, onde a alternância entre os modos de comando e inserção (tecla i) é pouco intuitiva, além da necessidade de memorizar comandos específicos para salvar e sair, como :wq. A ausência de interface gráfica e do uso do mouse exige maior precisão na digitação e na navegação pelas linhas de código, tornando a correção de erros de sintaxe no HTML e CSS um processo mais manual e rigoroso do que em editores convencionais.
